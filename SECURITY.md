# Security Policy

## Supported Versions

Security fixes are applied to the latest code on `main`.

## Reporting a Vulnerability

Please do not open public issues for sensitive vulnerabilities.

Report privately to the maintainer with:

1. Vulnerability summary
2. Reproduction steps
3. Impact assessment
4. Suggested mitigation (optional)

## Scope Notes

- This extension does not require user credentials.
- Avoid committing local machine paths or sensitive workspace data.
- Keep `.vscode/settings.json`, `.codex-notify`, and private artifacts out of shared commits unless intentionally required.
